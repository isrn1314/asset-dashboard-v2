# Codex資產網頁

## 專案簡介
這是一個用 HTML / CSS / JS 製作的個人資產總覽 / 資產管理小系統。

目前目標：
- 超實用
- 可長期使用
- 可持續擴充
- 手機可用
- 結構清楚、可維護

## 目前已有功能

- localStorage 儲存

## 資料儲存與匯入規則

### localStorage 使用的 key
- `asset_dashboard_state_v3`
  - 儲存主要資產資料狀態，包含帳戶、投資、UI 狀態與儲存時間資訊。
- `asset_dashboard_ui_v3`
  - 儲存目前使用中的分頁狀態。

### 目前資料來源
- 瀏覽器 `localStorage`
  - 平常操作後的資料，會儲存在目前瀏覽器的本機儲存空間。
- 使用者手動匯出 / 匯入 JSON 備份
  - 可由使用者自行匯出備份檔，之後再手動匯入回系統。

### 匯入資料的風險
- 匯入空資料可能覆蓋現有資料
  - 若匯入內容雖然格式上被接受，但實際資料為空，可能會直接覆蓋原本已存在的資料。
- 重複 `id` 可能導致編輯或刪除異常
  - 若匯入資料中有重複 `id`，可能造成編輯命中錯誤項目，或刪除時發生非預期結果。
- 清除瀏覽器資料會導致資料遺失
  - 目前資料主要依賴瀏覽器本機儲存，若清除瀏覽器站點資料或更換裝置，未備份資料可能遺失。

### 後續修正方向
- 匯入前必須更嚴格驗證
  - 需要確認必要欄位、資料型別與內容完整性，避免不完整資料直接覆蓋現況。
- 匯入前應檢查 `id` 唯一性
  - 需要在寫入前先確認所有資料項目的 `id` 不重複。
- 匯入前應保留現有資料備份
  - 在正式覆蓋前，應先保留當前資料，降低匯入失敗或匯錯檔案時的損失。
- 未來若修改 schema，必須同步更新 README
  - 只要資料結構、key、相容規則或備份格式有變動，就必須同步更新這份文件。
